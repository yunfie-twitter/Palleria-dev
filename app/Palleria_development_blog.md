# Palleria 開発日記

Palleria の開発ブログを公開します。  
Pixiv の作品を「読む」「探す」「保存する」までひとつの流れで扱えるようにするために、画面構成、保存処理、設定、セッション管理をまとめて整えました。

今回の更新では、見た目の調整だけでなく、日常利用で効く部分を中心に詰めています。

## 何を作っているか

Palleria は、Pixiv 向けのイラストビューアーです。  
ホーム、ランキング、検索、ブックマーク、小説、もっと見るを軸にしながら、作品を見つけた後の操作まで含めて完結させることを狙っています。

特に重視しているのは次の3点です。

- 作品を見つけた直後に、保存やブックマークへ自然につなげること
- 閲覧履歴や検索履歴を後から整理しやすくすること
- 端末内の保存状態やダウンロード進捗を把握しやすくすること

単に作品一覧を表示するだけではなく、使い続けるほど便利になる導線を作るのが Palleria の方針です。

## 今回の主な更新

### 保存フローの整理

保存まわりは、Palleria の中でも特に力を入れている領域です。

- 作者別フォルダ分け
- 作品別フォルダ分け
- 既定の保存場所の設定
- オフライン保存
- 保存済み作品の一覧化
- ダウンロード一覧の追加

保存したファイルが端末のどこに入るか、今どの処理が進行中かを追えるようにしています。  
ダウンロード一覧では、待機中・ダウンロード中・完了・失敗を状態別に分け、バックグラウンド処理をそのまま見える形にしました。

### 作品詳細の操作性改善

作品詳細では、保存、共有、URLコピー、ブラウザで開く、ミュートなどの操作をまとめています。  
さらに、保存時の確認ダイアログや長押し時の確認制御など、操作の粒度も調整できるようにしました。

加えて、ブックマーク時の連動動作も整理しています。

- 作者を自動フォロー
- タグを Watchlist 候補へ追加
- 保存と同時にブックマーク
- ブックマーク時の公開範囲を調整

作品を「見る」だけで終わらせず、あとで追い直せる状態へ自然に接続するための実装です。

### 検索と再訪の強化

検索は、イラストとユーザーを分けて扱っています。  
検索対象、期間、並び順、ブックマーク数などを切り替えられるようにし、用途ごとに結果を絞り込める構成にしています。

あわせて、検索履歴、最近見た作品、おすすめタグも用意しています。  
毎回ゼロから探し直すのではなく、再訪しやすい入口を増やすことを意識しています。

### 履歴・ミュート・データ管理

Palleria では、作品を見続けるほど効いてくる周辺機能もまとめて扱います。

- 閲覧履歴の保存
- 検索履歴の保存
- ユーザー・作品・タグのミュート
- Watchlist タグ
- お気に入りタグ
- データのエクスポート / インポート

履歴とミュートは、設定として単独で持つより、あとから整理できることのほうが重要です。  
そのため、データ管理画面でまとめて見直せるようにしています。

### アカウントと復帰導線

Pixiv 連携は、Web ログインと Refresh Token ログインの両方に対応しています。  
アカウントの追加方法を分けることで、導入時の選択肢を確保しました。

- Web ログイン
- Refresh Token ログイン
- アカウント切り替え
- ログアウト
- 認証切れ時の再ログイン案内

認証が切れたときは、単にエラーで止めるのではなく、再ログインへ戻れる流れを用意しています。  
セッションの寿命を前提にした復帰設計も、クライアントアプリでは重要だと考えています。

## 設定まわり

設定画面は、機能の一覧ではなく、使い方を調整するための入口としてまとめています。

- 一般設定
- 画像設定
- ブックマーク設定
- アカウント
- データ管理

一般設定では、テーマ、Dynamic Color、言語、R-18 表示、アニメーション、ハプティクス、起動時の画面などを扱います。  
画像設定では、画質、先読み、列数、プロキシ、保存先を調整できます。  
ブックマーク設定では、保存時の挙動を用途に応じて細かく変えられるようにしています。

設定項目が増えても迷いにくいよう、意味の近い項目を同じまとまりに置く構成を採っています。

## 技術的なポイント

Palleria は Jetpack Compose をベースに、Miuix のコンポーネント群を組み合わせて構成しています。  
実装では、見た目よりも状態管理と復帰性を優先しました。

意識しているポイントは次の通りです。

- 画面ごとの責務を分離する
- ViewModel に状態遷移を集約する
- 設定変更をアプリ全体へ一貫して反映する
- 保存や読み込みの進行状況を追跡できるようにする
- 認証切れや通信失敗から戻れる導線を残す

画像表示まわりでは、高品質画像、先読み、プロキシ差し替えにも対応しています。  
通信量と表示速度のバランスを調整しながら、実用優先で使える構成にしています。

## これから

今後は、保存後の作品管理や検索の再訪性をさらに強めていく予定です。  
また、一覧や遷移の反応速度、長く使う人向けの整理機能も引き続き詰めていきます。

Palleria は、毎日の Pixiv 利用の中で「あると助かる」を積み上げる方針で育てています。  
今後も、読む・探す・保存する、の流れを崩さずに改善を続けます。
