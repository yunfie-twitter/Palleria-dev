# Palleria 開発日記

前回は、Pixiv クライアントとして必要だった保存・検索・ブックマークまわりを中心にまとめました。  
今回はその続きとして、Palleria を「機能があるアプリ」から「毎日使い続けやすいアプリ」に近づけるために手を入れた部分を書きます。

Pixiv クライアントは、作品を見つけられるだけでは足りません。  
起動してすぐ使えること、途中で認証が切れても戻りやすいこと、保存したものや追いかけているシリーズにすぐ再訪できること。その積み重ねが、使い心地をかなり左右します。

今回の更新では、特に次の4点を重視しました。

- ログインまでの導線を迷いにくくすること
- 保存処理の進行状況を見えるようにすること
- Watchlist からシリーズへ再訪しやすくすること
- アプリを開かなくてもランキングを追えるようにすること

## 入口を整える

最初に見直したのは、使い始めの入口です。

Palleria では、Pixiv Web ログインと Refresh Token ログインの 2 つを使えるようにしています。  
ただ、選択肢が増えるほど、初見ではどちらを使えばいいか迷いやすくなります。そこで、ログイン画面を単なるボタン置き場ではなく、アプリで何ができるかを最初に短く伝える構成へ寄せました。

- 作品をすぐ見つける
- ブックマークをまとめる
- 続きからすぐ再開する

この 3 つをウォークスルーとして見せたうえで、下に Web ログインと Refresh Token ログインを並べています。  
いきなり設定や用語を並べるより、「このアプリで何が楽になるか」を先に見せるほうが、導入時の負担が小さくなると考えました。

また、ログイン画面には「このアプリケーションは Pixiv 公式とは一切関係ありません」という注意書きも明示しています。  
非公式クライアントであることは、設定の奥ではなく、使い始める時点で伝わるべき情報だからです。

## 認証が切れても戻りやすくする

日常利用では、ログイン成功よりも「認証が切れたあとにどう戻れるか」のほうが重要です。

Pixiv 連携は長く使っているとどうしても再認証が必要になる場面があります。  
そのときに、単に通信エラーで止まるだけだと、利用者から見ると何が起きたのか分かりづらくなります。

そこで Palleria では、認証切れを検知したときに再ログイン導線へ戻せる構成をより明確にしました。  
前回の記事でも復帰性は重視していましたが、今回は「ログインまでの入口」と「認証切れ後の戻り先」が同じ文脈でつながるようにしています。

機能が増えてくるほど、異常時の戻り先が曖昧なアプリは使いづらくなります。  
Palleria では、正常時の快適さだけでなく、途切れたときに復帰できることも同じくらい重要な要件として扱っています。

## 保存処理を見える化する

前回、保存フロー自体はかなり整えました。  
その次に必要だったのは、「今どうなっているか」を確認できる画面です。

そこで追加したのが、ダウンロード一覧です。

- すべて
- ダウンロード中
- 完了

というタブで状態を切り替えながら、保存した順に一覧できるようにしました。  
各項目には、待機中、ダウンロード中、完了、失敗の状態をそのまま表示しています。

保存機能は、押した瞬間だけ快適でも不十分です。  
複数枚の作品や連続した保存を扱うようになると、裏で何が進んでいるかが見えないだけで不安になります。

特に非公式クライアントでは、OS 側の制約や通信状態の揺れも含めて、「保存したはずなのに今どうなっているのか」が気になる場面が増えます。  
ダウンロード一覧は、そうした不透明さを減らすための画面です。

## Watchlist を「あとで見る」で終わらせない

Palleria では、Watchlist を単なるタグの保管場所ではなく、再訪の起点として育てています。

これまでも Watchlist タグ自体は扱っていましたが、今回はそこから漫画シリーズへ入り直しやすい専用画面を用意しました。  
保存したシリーズをカードで並べて、サムネイル、タイトル、ページ数を見ながら直接開ける構成です。

さらに、

- Pull to Refresh
- 自動読み込み
- 手動の「さらに読み込む」
- 空状態やエラー状態の表示

まで含めて、一覧画面としての完成度を上げています。

Pixiv で気になったシリーズは、その場で読むだけでなく、あとでまとめて追い直したくなります。  
そのときに、タグ名だけを手掛かりにするのではなく、シリーズ単位で見返せるほうがずっと自然です。

前回の記事では「保存」「検索」「ブックマーク」をつなぐ話を書きましたが、今回はその先にある「再訪」の質を上げた形になります。

## ホーム画面からランキングを追えるようにする

アプリを起動しなくても作品に触れられる入口として、ランキングウィジェットも用意しました。

このウィジェットでは、ホーム画面上にランキング作品を表示し、更新や作品遷移まで行えるようにしています。  
ログインしていない場合は、その場でログイン導線を出し、ログイン済みなら最新のランキングを確認できる構成です。

Palleria の本体が担うのは、深く見るための画面遷移です。  
一方でウィジェットは、「今日は何が上がっているか」を一瞬で掴むための入口として機能します。

Pixiv の利用は、必ずしも毎回検索から始まるわけではありません。  
日によっては、まずランキングをざっと眺めて、気になる作品だけ開きたいこともあります。そうした軽い接触点を増やす意味でも、ウィジェットの価値は大きいと感じています。

## 何を優先しているか

今回の改善を通して、Palleria で重視している方向性はかなりはっきりしてきました。

- 最初の一歩で迷わせないこと
- 途中で途切れても復帰しやすいこと
- 保存や追跡の状態が見えること
- 後日また開くときの入口を増やすこと

どれも派手な新機能ではありません。  
ただ、こういう部分が整っていないと、保存や検索のような主機能も実際には使い続けづらくなります。

非公式 Pixiv クライアントを作るうえでは、API を叩いて画面を増やすだけではなく、「毎日触っても疲れにくい導線」をどこまで作れるかが大事だと考えています。

## これから

次は、再訪性をさらに強めるために、一覧同士のつながりや履歴まわりの整理をもう少し詰めていく予定です。  
また、保存した作品やブックマーク済み作品を、あとから扱いやすくするための見せ方も引き続き改善していきます。

Palleria は、Pixiv を毎日見る人のための非公式クライアントとして、ひとつひとつの導線を地道に磨いています。  
前回が「見つけて保存するまで」の話だとしたら、今回は「また戻ってきたくなる形に整える」話でした。

---

Palleria は Pixiv 公式とは無関係の非公式クライアントです。
